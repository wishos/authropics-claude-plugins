# 学习样式插件

此插件将未发布的"学习"输出样式与解释性功能结合作为 SessionStart hook。

注意：此插件与原始未发布的"学习"输出样式的不同之处在于，它还包含 [explanatory-output-style 插件](https://github.com/anthropics/claude-code/tree/main/plugins/explanatory-output-style) 的所有功能，提供交互式学习和教育性见解。

警告：除非您愿意承担此插件额外指令的 token 消耗和学习模式的交互特性，否则不要安装此插件。

## 功能

启用后，此插件自动在每个会话开始时添加指令，鼓励 Claude：

1. **学习模式：** 在决策点请求有意义的代码贡献来让您参与主动学习
2. **解释模式：** 提供关于实现选择和代码库模式的教育性见解

Claude 不会自动实现一切，而是：

1. 识别您可以编写 5-10 行有意义代码的机会
2. 专注于您真正重要的业务逻辑和设计选择
3. 为您的贡献准备上下文和位置
4. 解释权衡并指导您的实现
5. 在编写代码之前和之后提供教育性见解

## 工作原理

该插件使用 SessionStart hook 将额外上下文注入每个会话。此上下文指示 Claude 采用交互式教学方法，让您在编写关键代码部分时积极参与。

## Claude 何时请求贡献

Claude 将要求您编写代码用于：
- 有多种有效方法的业务逻辑
- 错误处理策略
- 算法实现选择
- 数据结构决策
- 用户体验决策
- 设计模式和架构选择

## Claude 何时不请求贡献

Claude 将直接实现：
- 样板或重复代码
- 没有有意义选择的明显实现
- 配置或设置代码
- 简单的 CRUD 操作

## 示例交互

**Claude：** 我已设置好认证中间件。会话超时行为是一个安全与用户体验的权衡——会话应该在活动时自动延长，还是有硬性超时？

在 `auth/middleware.ts` 中实现 `handleSessionTimeout()` 函数来定义超时行为。

考虑：自动延长改善用户体验但可能使会话保持更长时间；硬性超时更安全但可能会让活跃用户感到沮丧。

**您：** [编写 5-10 行实现您首选的方法]

## 教育性见解

除了交互式学习外，Claude 还将使用以下格式提供关于实现选择的教育性见解：

```
`★ Insight ─────────────────────────────────────`
[关于代码库或实现的 2-3 个关键教育要点]
`─────────────────────────────────────────────────`
```

这些见解专注于：

- 代码库的具体实现选择
- 代码中的模式和约定
- 权衡和设计决策
- 代码库特定的细节，而不是通用编程概念

## 使用方法

安装后，插件在每个会话开始时自动激活。无需额外配置。

## 从输出样式迁移

此插件将未发布的"学习"输出样式与已弃用的"解释性"输出样式相结合。它提供交互式学习体验，让您在有意义的决策点主动贡献代码，同时接收关于实现选择的教育性见解。

如果您以前使用过 explanatory-output-style 插件，此学习插件包含该所有功能加上交互式学习功能。

此 SessionStart hook 模式大致相当于 CLAUDE.md，但它更灵活，允许通过插件分发。

## 管理更改

- 禁用插件 - 保留代码安装在您的设备上
- 卸载插件 - 从您的设备删除代码
- 更新插件 - 创建此插件的本地副本以个性化此插件
  - 提示：让 Claude 阅读 https://docs.claude.com/zh-CN/docs/claude-code/plugins.md 并为您设置！

## 理念

边做边学比被动观察更有效。此插件将您与 Claude 的交互从"观看和学习"转变为"构建和理解"，确保您通过有意义的逻辑实践编码来发展实用技能。
