# 解释性输出样式插件

此插件将已弃用的解释性输出样式作为 SessionStart hook 重新实现。

警告：除非您愿意承担此插件额外指令的 token 消耗和输出，否则不要安装此插件。

## 功能

启用后，此插件自动在每个会话开始时添加指令，鼓励 Claude：

1. 提供关于实现选择的教育性见解
2. 解释代码库模式和决策
3. 在任务完成和学习机会之间取得平衡

## 工作原理

该插件使用 SessionStart hook 将额外上下文注入每个会话。此上下文指示 Claude 在编写代码之前和之后提供简短的教育性解释，格式如下：

```
`★ Insight ─────────────────────────────────────`
[2-3 个关键教育要点]
`─────────────────────────────────────────────────`
```

## 使用方法

安装后，插件在每个会话开始时自动激活。无需额外配置。

见解专注于：

- 代码库的具体实现选择
- 代码中的模式和约定
- 权衡和设计决策
- 代码库特定的细节，而不是通用编程概念

## 从输出样式迁移

此插件替换已弃用的"解释性"输出样式设置。如果您以前使用：

```json
{
  "outputStyle": "Explanatory"
}
```

您现在可以通过安装此插件来获得相同的行为。

更一般地说，此 SessionStart hook 模式大致相当于 CLAUDE.md，但它更灵活，允许通过插件分发。

注意：涉及软件开发展开任务以外的输出样式最好表示为 [subagent](https://docs.claude.com/zh-CN/docs/claude-code/sub-agents)，而不是 SessionStart hook。Subagent 更改系统提示，而 SessionStart hook 添加到默认系统提示。

## 管理更改

- 禁用插件 - 保留代码安装在您的设备上
- 卸载插件 - 从您的设备删除代码
- 更新插件 - 创建此插件的本地副本以进行个性化
  - 提示：让 Claude 阅读 https://docs.claude.com/zh-CN/docs/claude-code/plugins.md 并为您设置！
