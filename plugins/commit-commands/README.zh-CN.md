# 提交命令插件 (Commit Commands Plugin)

使用简单的命令自动化 git 工作流，包括提交、推送和创建 pull request。

## 概述

提交命令插件自动化常见的 git 操作，减少上下文切换和手动命令执行。无需运行多个 git 命令，使用单个斜杠命令即可处理整个工作流。

## 命令

### `/commit`

根据已暂存和未暂存的变更自动生成提交消息创建 git 提交。

**功能：**
1. 分析当前 git 状态
2. 检查已暂存和未暂存的变更
3. 检查最近的提交消息以匹配您的仓库风格
4. 起草适当的提交消息
5. 暂存相关文件
6. 创建提交

**用法：**
```bash
/commit
```

**工作流程示例：**
```bash
# 对代码进行一些更改
# 然后简单运行：
/commit

# Claude 将：
# - 审查您的更改
# - 暂存文件
# - 创建带有适当消息的提交
# - 显示提交状态
```

**功能特点：**
- 自动起草符合您仓库风格的提交消息
- 遵循 conventional commit 规范
- 避免提交包含密钥的文件（.env, credentials.json）
- 在提交消息中包含 Claude Code 归属

### `/commit-push-pr`

一步完成提交、推送和创建 pull request 的完整工作流命令。

**功能：**
1. 创建新分支（如果当前在 main 上）
2. 暂存并提交带有适当消息的变更
3. 将分支推送到 origin
4. 使用 `gh pr create` 创建 pull request
5. 提供 PR 链接

**用法：**
```bash
/commit-push-pr
```

**工作流程示例：**
```bash
# 进行您的更改
# 然后运行：
/commit-push-pr

# Claude 将：
# - 创建功能分支（如需要）
# - 提交您的更改
# - 推送到远程
# - 打开带有摘要和测试计划的 PR
# - 给您 PR 链接进行审查
```

**功能特点：**
- 分析分支中的所有提交（不仅是最新一个）
- 创建全面的 PR 描述，包括：
  - 变更摘要（1-3 个要点）
  - 测试计划清单
  - Claude Code 归属
- 自动处理分支创建
- 使用 GitHub CLI (`gh`) 创建 PR

**要求：**
- 必须安装并认证 GitHub CLI (`gh`)
- 仓库必须有名为 `origin` 的 remote

### `/clean_gone`

清理已从远程仓库删除的本地分支。

**功能：**
1. 列出所有本地分支以识别 [gone] 状态
2. 识别并删除与 [gone] 分支关联的 worktree
3. 删除所有标记为 [gone] 的分支
4. 提供关于已删除分支的反馈

**用法：**
```bash
/clean_gone
```

**工作流程示例：**
```bash
# PR 合并并删除远程分支后
/clean_gone

# Claude 将：
# - 找到所有标记为 [gone] 的分支
# - 删除任何关联的 worktree
# - 删除过时的本地分支
# - 报告清理了哪些内容
```

**功能特点：**
- 同时处理常规分支和 worktree 分支
- 在删除分支之前安全地删除 worktree
- 显示关于已删除内容的清晰反馈
- 如果不需要清理则报告

**使用场景：**
- 合并并删除远程分支后
- 当您的本地分支列表充满过时的分支时
- 在常规仓库维护期间

## 安装

此插件包含在 Claude Code 仓库中。使用 Claude Code 时命令自动可用。

## 最佳实践

### 使用 `/commit`
- 提交前审查暂存的变更
- 让 Claude 分析您的更改并匹配您仓库的提交风格
- 信任自动生成的消息，但验证其准确性
- 在开发期间用于常规提交

### 使用 `/commit-push-pr`
- 当您准备好创建 PR 时使用
- 确保您所有的更改都已完成并经过测试
- Claude 将分析完整的分支历史以获取 PR 描述
- 审查 PR 描述并在需要时进行编辑
- 当您想最小化上下文切换时使用

### 使用 `/clean_gone`
- 定期运行以保持分支列表整洁
- 特别是在合并多个 PR 后有用
- 安全运行 - 仅删除远程已删除的分支
- 帮助维护整洁的本地仓库

## 工作流集成

### 快速提交工作流：
```bash
# 编写代码
/commit
# 继续开发
```

### 功能分支工作流：
```bash
# 跨多个提交开发功能
/commit  # 第一次提交
# 更多更改
/commit  # 第二次提交
# 准备好创建 PR
/commit-push-pr
```

### 维护工作流：
```bash
# 几个 PR 合并后
/clean_gone
# 清理工作区准备下一个功能
```

## 要求

- 必须安装并配置 Git
- 对于 `/commit-push-pr`：必须安装并认证 GitHub CLI (`gh`)
- 仓库必须是带有 remote 的 git 仓库

## 故障排除

### `/commit` 创建空提交

**问题**：没有要提交的更改

**解决方案**：
- 确保您有未暂存或已暂存的更改
- 运行 `git status` 验证更改存在

### `/commit-push-pr` 创建 PR 失败

**问题**：`gh pr create` 命令失败

**解决方案**：
- 安装 GitHub CLI：`brew install gh`（macOS）或参阅 [GitHub CLI 安装](https://cli.github.com/)
- 认证：`gh auth login`
- 确保仓库有 GitHub remote

### `/clean_gone` 找不到分支

**问题**：没有标记为 [gone] 的分支

**解决方案**：
- 运行 `git fetch --prune` 更新远程跟踪
- 分支必须从远程删除才会显示为 [gone]

## 提示

- **与其他工具结合使用**：在开发期间使用 `/commit`，然后在准备好时使用 `/commit-push-pr`
- **让 Claude 起草消息**：提交消息分析会从您仓库的风格中学习
- **定期清理**：每周运行 `/clean_gone` 以保持分支列表整洁
- **推送前审查**：始终在推送前审查提交消息和更改
